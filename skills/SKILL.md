# Blog Management Skill

Manage and maintain the Venant Blog (Hugo) infrastructure using the consolidated single-repo, dual-branch architecture.

## Architecture

The blog operates on a single repository (`farizvect/venant-blog`) with two distinct branches:
- **`templ`**: J jantung (heart) of the blog. Contains Hugo source code, themes (submodules), and content.
- **`main`**: The deployment target. Contains only static HTML/CSS/JS generated by Hugo.
- **CI/CD**: GitHub Actions builds the `templ` branch and pushes the output to `main`.

## Initial Setup Guide

### 1. Requirements
Install Hugo Extended (required for PaperMod/SCSS):
```bash
sudo apt update
sudo apt install hugo
```

### 2. Repository Initialization
```bash
mkdir VENANT-BLOG && cd VENANT-BLOG
git init
hugo new site .
git checkout -b templ
git submodule add --force https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
```

### 3. GitHub Authentication (PAT)
To allow agentic pushing, use a Personal Access Token (PAT):
```bash
# Configure git to store credentials
git config --global credential.helper store
# Trigger login prompt once by pushing
gh auth login --with-token < your_pat_file.txt
```

## Deployment Automation

The blog uses GitHub Actions. The workflow file must exist at `.github/workflows/gh-pages.yml` in the `templ` branch.
See [references/deployment-workflow.yml](references/deployment-workflow.yml).

## Daily Workflow

### Creating a New Post
1. **Write**: Create `.md` in `content/posts/`.
2. **Cleanup**: Run `stop-slop` skill on the draft.
3. **Visualize**: Generate cover using `grok-image-gen` skill.
4. **Deploy**:
   ```bash
   git add .
   git commit -m "feat(post): <title>"
   git push origin templ
   ```
